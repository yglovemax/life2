# Phase 1 Roadmap

## Step 1: 可运行骨架

状态：已开始。

交付：

- 独立仓库和项目结构。
- 原版需求文档归档。
- FastAPI 服务。
- SQLite 数据库。
- 两个一期页面和 31 个模块的预置数据。
- 模块中心 UI。
- 模块详情 UI。
- 单模块测试调用和追踪记录。

## Step 2: 模块配置闭环

状态：已完成第一版。

交付：

- 新增模块。
- 编辑模块基础信息。
- 编辑 Prompt 五段式。
- 编辑字段契约。
- 编辑 Fallback 内容。
- 保存草稿。

## Step 3: 测试中心

状态：已完成第一版。

交付：

- 选择页面、模块、测试用户、日期、模型。
- 单模块测试。
- 多模块批量测试。
- 修复前后结果对比。
- 人工评分。

## Step 4: 知识库

交付：

- 上传 Markdown。
- 人工录入知识条目。
- 标签管理。
- 轻量检索。
- 生成命中记录。

## Step 5: 成本、Fallback、发布

交付：

- 成本总览和明细。
- 按页面、模块、模型统计。
- Fallback 触发原因和告警。
- 模块状态流转。
- 灰度、上线、回滚。

## Step 6: App 后端对接

交付：

- 页面级 JSON API。
- 模块级 JSON API。
- 请求追踪 ID。
- 接口鉴权。
- 正式调用和测试调用隔离。
